DOCUMENTAÇÃO DO PROJETO POLYDEV

1. Objetivo do Projeto

O projeto POLYDEV tem como objetivo criar um portal de conteúdos de programação, permitindo disponibilizar arquivos, downloads, pacotes ZIP, ranking de downloads e administração dos conteúdos utilizando apenas GitHub Pages e Supabase.

2. Arquitetura do Sistema

O sistema POLYDEV funciona sem servidor próprio, utilizando apenas frontend em HTML e JavaScript e banco de dados no Supabase.
USUÁRIO
↓
GitHub Pages (HTML + JavaScript)
↓
Supabase (Banco de Dados)
↓
Arquivos Downloads / ZIP
↓
Ranking / Estatísticas

4. Conceito de Categoria

No projeto POLYDEV, categoria não é o arquivo ZIP. Categoria representa a linguagem ou extensão dos arquivos.
python → arquivos .py
html → arquivos .html
sql → arquivos .sql
cpp → arquivos .cpp
js → arquivos .js

5. Banco de Dados

O sistema utiliza o Supabase como banco de dados. Existem duas tabelas principais:
Tabela categorias → ranking de downloads
Tabela conteudos → arquivos individuais

6. Páginas do Sistema

index.html → Menu principal
top.html → Ranking downloads
categorias.html → Lista categorias
conteudos.html → Lista conteúdos
buscar.html → Busca conteúdos
admin.html → Administração
guia.html → Documentação
config.js → Configuração Supabase
