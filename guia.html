POLYDEV Documentação e Guia do Projeto

1. Visão Geral do Projeto

O projeto POLYDEV é uma plataforma web para disponibilizar códigos, exemplos, templates, scripts e materiais de programação para download. A ideia do projeto é funcionar como uma biblioteca de códigos organizada por linguagem, categoria e popularidade.

O objetivo não é apenas armazenar arquivos, mas criar uma plataforma educacional e uma biblioteca técnica para programadores e estudantes.

2. Arquitetura do Sistema

O POLYDEV utiliza uma arquitetura moderna chamada serverless, ou seja, não existe um servidor tradicional rodando backend.

A arquitetura funciona da seguinte forma:

GitHub Pages (Frontend)
↓
HTML + JavaScript
↓
Supabase (API REST)
↓
PostgreSQL (Banco de Dados)
↓
Arquivos armazenados no GitHub (Downloads)

Isso permite que a plataforma funcione sem servidor próprio, sem VPS e com custo praticamente zero.

3. Tecnologias Utilizadas

O projeto utiliza as seguintes tecnologias:

GitHub - armazenamento dos arquivos e hospedagem do site
GitHub Pages - hospedagem do frontend
Supabase - API e backend serverless
PostgreSQL - banco de dados
HTML - estrutura das páginas
CSS - layout e visual
JavaScript - integração com o banco de dados DBeaver ferramenta gráfica universal open source para bancos de dados.

6. Fluxo de funcionamento da biblioteca

O funcionamento do sistema ocorre da seguinte forma:

Usuário entra na página conteudos.html
↓
O sistema consulta o banco de dados
↓
Lista automaticamente os conteúdos disponíveis
↓
Usuário clica em Download
↓
O sistema registra o download no banco
↓
O arquivo abre para download
↓
O contador de downloads é atualizado

Esse fluxo é o núcleo do sistema POLYDEV.

10. Objetivo final do POLYDEV

O objetivo final do projeto POLYDEV é se tornar uma plataforma de ensino, biblioteca de códigos e portal educacional voltado para programação, bancos de dados e desenvolvimento de software.

A ideia é centralizar códigos, exemplos, aulas, guias e materiais em um único ambiente organizado por tecnologia.

11. Conclusão

O projeto POLYDEV já possui a parte mais difícil concluída: arquitetura, banco de dados, integração com API, catálogo automático e sistema de downloads.

As próximas etapas agora são evoluções da interface, organização dos conteúdos e funcionalidades de usuários.

O projeto já pode ser considerado uma plataforma funcional de biblioteca de códigos.
