Embedded SQL with Knex.js

JavaScript Knex.js SQL PostgreSQL Backend

Concept

Knex.js allows JavaScript developers to build SQL queries programmatically while keeping the power of relational databases, transactions, joins, aggregations and query composition.

This example simulates a real POLYDEV backend routine: generating a ranking of downloaded contents using joins, grouping, transaction control and system logging.

Advanced JavaScript + Knex.js Example

// polydev_knex_advanced.js
// JavaScript + Knex.js | Embedded SQL Style

import knex from "knex";

const db = knex({
  client: "pg",
  connection: {
    host: "localhost",
    port: 5432,
    user: "postgres",
    password: "postgres",
    database: "polydev"
  }
});

async function gerarRankingDownloads() {
  try {
    const resultado = await db.transaction(async trx => {

      const ranking = await trx("downloads as d")
        .join("conteudos as c", "c.id", "d.conteudo_id")
        .select(
          "c.categoria",
          "c.titulo"
        )
        .count("d.id as total_downloads")
        .max("d.created_at as ultimo_download")
        .where("c.ativo", true)
        .groupBy("c.categoria", "c.titulo")
        .orderBy("total_downloads", "desc")
        .limit(10);

      await trx("logs_sistema").insert({
        evento: "RANKING_DOWNLOADS_GERADO",
        detalhe: "Top 10 conteúdos calculados com Knex.js",
        created_at: trx.fn.now()
      });

      return ranking;
    });

    console.table(resultado);

  } catch (error) {
    console.error("Erro ao gerar ranking:", error.message);

  } finally {
    await db.destroy();
  }
}

gerarRankingDownloads();

Why this matters

This pattern is useful when JavaScript applications need strong database interaction without writing raw SQL everywhere.

Knex.js works as a bridge between application logic and relational database power. It keeps the code expressive, reusable and closer to real backend architecture.

POLYDEV Insight

Embedded SQL inside JavaScript is not just syntactic convenience. It is a practical way to combine application flow, business rules and database intelligence in the same backend pipeline.
