Artigo 02 de 05 — Sensores e Estados com Classes Java

Visão Geral

Neste segundo artigo da série Embedded Java Bitwise, vamos trabalhar com duas classes muito úteis para sistemas reais:

  • SensorStatus.java — controle de status de sensores
  • WorkflowState.java — combinação de estados de workflow

O foco aqui é mostrar como bits podem representar condições simultâneas, falhas, alertas e estados de processamento.

Classe 03 — SensorStatus.java

Esta classe simula sensores em um sistema embarcado. Cada bit representa um tipo de leitura ou condição operacional. 

public class SensorStatus {

    private static final int TEMPERATURE_OK = 1;   // 0001
    private static final int PRESSURE_OK    = 2;   // 0010
    private static final int HUMIDITY_OK    = 4;   // 0100
    private static final int BATTERY_LOW    = 8;   // 1000

    private int status;

    public SensorStatus() {
        this.status = 0;
    }

    public void activate(int flag) {
        status |= flag;
    }

    public void deactivate(int flag) {
        status &= ~flag;
    }

    public boolean isActive(int flag) {
        return (status & flag) != 0;
    }

    public void showBinary() {
        System.out.println(
            "STATUS: " +
            String.format("%4s", Integer.toBinaryString(status))
            .replace(' ', '0')
        );
    }

    public void showStatus() {
        System.out.println("TEMPERATURE_OK : " + isActive(TEMPERATURE_OK));
        System.out.println("PRESSURE_OK    : " + isActive(PRESSURE_OK));
        System.out.println("HUMIDITY_OK    : " + isActive(HUMIDITY_OK));
        System.out.println("BATTERY_LOW    : " + isActive(BATTERY_LOW));
    }

    public static void main(String[] args) {

        SensorStatus sensor = new SensorStatus();

        System.out.println("=== SENSOR NORMAL ===");

        sensor.activate(TEMPERATURE_OK);
        sensor.activate(PRESSURE_OK);
        sensor.activate(HUMIDITY_OK);

        sensor.showBinary();
        sensor.showStatus();

        System.out.println();

        System.out.println("=== BATTERY WARNING ===");

        sensor.activate(BATTERY_LOW);
        sensor.deactivate(PRESSURE_OK);

        sensor.showBinary();
        sensor.showStatus();
    }
}

Classe 04 — WorkflowState.java

Esta classe mostra como múltiplos estados de workflow podem coexistir dentro de uma única variável inteira. 

public class WorkflowState {

    private static final int CREATED   = 1;    // 00001
    private static final int VALIDATED = 2;    // 00010
    private static final int APPROVED  = 4;    // 00100
    private static final int SENT      = 8;    // 01000
    private static final int ARCHIVED  = 16;   // 10000

    private int state;

    public WorkflowState() {
        this.state = 0;
    }

    public void addState(int flag) {
        state |= flag;
    }

    public void removeState(int flag) {
        state &= ~flag;
    }

    public boolean hasState(int flag) {
        return (state & flag) != 0;
    }

    public void showBinary() {
        System.out.println(
            "STATE: " +
            String.format("%5s", Integer.toBinaryString(state))
            .replace(' ', '0')
        );
    }

    public void showStates() {
        System.out.println("CREATED   : " + hasState(CREATED));
        System.out.println("VALIDATED : " + hasState(VALIDATED));
        System.out.println("APPROVED  : " + hasState(APPROVED));
        System.out.println("SENT      : " + hasState(SENT));
        System.out.println("ARCHIVED  : " + hasState(ARCHIVED));
    }

    public static void main(String[] args) {

        WorkflowState order = new WorkflowState();

        System.out.println("=== WORKFLOW START ===");

        order.addState(CREATED);
        order.addState(VALIDATED);

        order.showBinary();
        order.showStates();

        System.out.println();

        System.out.println("=== WORKFLOW APPROVED AND SENT ===");

        order.addState(APPROVED);
        order.addState(SENT);

        order.showBinary();
        order.showStates();

        System.out.println();

        System.out.println("=== ARCHIVING ===");

        order.addState(ARCHIVED);
        order.removeState(SENT);

        order.showBinary();
        order.showStates();
    }
}

Resumo Técnico

status |= flag;       // Ativa um estado
status &= ~flag;      // Remove um estado
(status & flag) != 0  // Verifica se o estado existe

Esse padrão é muito útil quando vários estados podem coexistir ao mesmo tempo, sem necessidade de múltiplas variáveis booleanas.
