🎯 Objetivo

Aprender a observar um executável como artefato técnico: identificar metadados, extrair strings, visualizar seções, desmontar instruções e iniciar uma análise estática controlada.

📖 Conceitos principais

  • Diferença entre código-fonte e binário.
  • Assembly em arquitetura x86_64.
  • Análise estática: observar sem executar.
  • Análise dinâmica: observar durante a execução.

🖥 Comandos práticos

# Informações básicas
file crackme.elf
strings crackme.elf | head -n 50
strings crackme.elf | grep -E "password|senha|flag|key|secret|admin"

# Desmontagem
objdump -d -M intel crackme.elf > disassembly.txt
objdump -h crackme.elf

🧰 Análise com radare2

# Abrir binário com análise automática
r2 -A crackme.elf

# Dentro do radare2:
aaa          # análise automática
afl          # listar funções
pdf @ main   # ver função main
/ senha      # buscar strings
VV           # modo visual

🧪 Ferramentas recomendadas

  • radare2 — análise em linha de comando.
  • Cutter — interface gráfica para radare2.
  • Ghidra — suíte gratuita de engenharia reversa.

🛡 Defesa

Para dificultar análise indevida em software real, use técnicas como strip, ofuscação e packing. Use packing com cuidado, porque pode gerar falsos positivos em antivírus e dificultar suporte legítimo.

💡 Dica POLYDEV

Engenharia reversa não é magia. É observação disciplinada. Primeiro descubra o tipo do arquivo, depois procure strings, depois entenda seções, funções e fluxo. Brick on brick.

🧪 Exercício

Baixe um crackme iniciante em ambiente de laboratório autorizado e tente descobrir a senha usando strings e radare2. O objetivo é aprender leitura de binários, não atacar sistemas reais.
