🧠 Situação Atual do Projeto

O projeto POLYDEV evoluiu de uma ideia educacional para uma aplicação funcional baseada em arquitetura moderna e leve. Hoje o sistema opera sem backend dedicado, utilizando serviços cloud e integração direta via API.

Arquitetura implementada

  • Frontend: GitHub Pages (HTML + JavaScript)
  • API: Supabase REST
  • Banco de Dados: PostgreSQL
  • Administração: DBeaver
  • Modelo: Serverless

⚙️ Funcionalidades já implementadas

  • ✔ Cadastro persistente de usuários integrado ao Supabase
  • ✔ Sistema universal de conteúdos baseado em categorias
  • ✔ Viewers especializados para linguagens e bancos
  • ✔ Visualizador dinâmico de códigos fonte
  • ✔ Download individual de programas e guias
  • ✔ Contador de downloads integrado ao PostgreSQL
  • ✔ Integração completa com GitHub Pages
  • ✔ Navegação dinâmica entre Home, Menus e Viewers
  • ✔ Estrutura preparada para múltiplas linguagens
  • ✔ Sistema responsivo para desktop e mobile
  • ✔ Menus isométricos interativos
  • ✔ Integração direta com arquivos reais hospedados no GitHub

📊 Sistema Analítico (Destaque)

Um dos pontos mais importantes do projeto foi a criação de um sistema de análise de dados interno.

  • Tabela analítica: dashgeral
  • Atualização automática via trigger
  • Agregação por categoria, conteúdo e usuário
  • Leitura simplificada via API REST
  • Métricas reais de utilização por categoria
  • Rastreamento de downloads por conteúdo
  • Estrutura preparada para ranking TOP downloads
  • Consolidação estatística por linguagem
  • Integração analítica entre conteúdos e usuários

Isso transformou o projeto em algo além de um site — tornou-se uma pequena plataforma de dados.

🚀 Avaliação Técnica

  • Arquitetura: ✔ Moderna (Serverless)
  • Banco de dados: ✔ Bem estruturado
  • Integração API: ✔ Funcional
  • Automação (Trigger): ✔ Profissional
  • Frontend: ✔ Leve e eficiente

A solução adotada (tabela materializada + trigger) foi uma decisão estratégica muito sólida, priorizando estabilidade, simplicidade e controle.
