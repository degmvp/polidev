Apache Cassandra – LAB 06
Cassandra × SQL

Este laboratório apresenta um comparativo direto entre Apache Cassandra e bancos relacionais SQL, destacando diferenças conceituais, técnicas e de uso real.

Objetivo do LAB
  • Comparar conceitos fundamentais
  • Entender quando usar Cassandra
  • Entender quando usar SQL
  • Fixar o mindset NoSQL

1️⃣ Visão Geral

Aspecto Cassandra SQL
Modelo Wide-column / NoSQL Relacional
Escalabilidade Horizontal (scale-out) Vertical (scale-up)
Schema Flexível Rígido
Alta disponibilidade Nativa Depende de configuração

2️⃣ Escrita de Dados

Cassandra
Alta taxa de escrita, distribuída e paralela. 
INSERT INTO clientes_por_cidade (cidade, id, nome)
VALUES ('Recife', uuid(), 'Ana Silva');
SQL
Escrita transacional com bloqueios e controle ACID. 
INSERT INTO clientes (nome, cidade)
VALUES ('Ana Silva', 'Recife');

3️⃣ Leitura de Dados

Cassandra
Leitura rápida quando a tabela foi modelada corretamente. 
SELECT * FROM clientes_por_cidade
WHERE cidade = 'Recife';
SQL
Leitura flexível usando JOINs e filtros variados. 
SELECT c.nome, p.nome
FROM clientes c
JOIN pedidos p ON p.cliente_id = c.id;

4️⃣ JOINs

Recurso Cassandra SQL
JOIN ❌ Não existe ✔ Totalmente suportado
Subqueries ❌ Não ✔ Sim

5️⃣ Consistência

Cassandra
Consistência eventual (configurável por nível).
SQL
Consistência forte (ACID).

6️⃣ Casos de Uso

Cassandra SQL
Logs Financeiro
IoT ERP
Eventos CRM
Alta escrita Relatórios complexos

7️⃣ Erros Comuns

  • Tentar usar Cassandra como SQL
  • Esperar JOINs
  • Usar índices indiscriminadamente

8️⃣ Regra Final

Use Cassandra quando:
  • Alta disponibilidade é crítica
  • Escalabilidade horizontal é necessária
  • Modelo de leitura é previsível
Use SQL quando:
  • Relacionamentos complexos
  • Consultas dinâmicas
  • Consistência forte

9️⃣ Encerramento

  • Você concluiu o guia Cassandra
  • Entendeu diferenças estruturais
  • Adotou o mindset correto
