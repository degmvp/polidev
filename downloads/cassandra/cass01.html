Apache Cassandra – LAB 01
CQL Básico

Este laboratório introduz os comandos fundamentais da linguagem CQL (Cassandra Query Language), utilizando o DataStax Astra DB ou um cluster Cassandra padrão.

Objetivo do LAB
  • Criar Keyspace
  • Criar Tabela
  • Inserir Dados
  • Consultar Dados

1️⃣ Conectando ao CQL

No Astra DB, utilize o CQL Console.
Em ambiente local, utilize: 

cqlsh

2️⃣ Criando o Keyspace

O Keyspace equivale conceitualmente a um database em bancos relacionais. 

CREATE KEYSPACE labks
WITH replication = {
  'class': 'NetworkTopologyStrategy',
  'datacenter1': 3
};
Observação:
No Astra DB, o Keyspace geralmente já vem criado. Use este comando apenas para fins didáticos.

3️⃣ Selecionando o Keyspace

USE labks;

4️⃣ Criando a Tabela

Tabela simples para armazenar clientes. 

CREATE TABLE clientes (
    id UUID PRIMARY KEY,
    nome   text,
    email  text,
    cidade text
);
Chave Primária
No Cassandra, a chave primária define:
  • Distribuição dos dados
  • Particionamento
  • Ordem física

5️⃣ Inserindo Dados

INSERT INTO clientes (id, nome, email, cidade)
VALUES (uuid(), 'Ana Silva', 'ana@email.com', 'Recife');

INSERT INTO clientes (id, nome, email, cidade)
VALUES (uuid(), 'Carlos Lima', 'carlos@email.com', 'São Paulo');

6️⃣ Consultando Dados

SELECT * FROM clientes;

Resultado esperado:

 id                                   | cidade      | email             | nome
--------------------------------------+-------------+-------------------+------------
 36b32b03-6a21-4fcf-b667-523051615132 | Recife      | ana@email.com     | Ana Silva

7️⃣ Conceitos Importantes

  • Cassandra NÃO suporta JOIN
  • Consultas devem ser planejadas antes da modelagem
  • SELECT sem WHERE só é aceitável em LAB
  • Escalabilidade horizontal é o foco

8️⃣ Encerramento do LAB

  • Executou comandos CQL básicos
  • Entendeu a estrutura mínima de um banco Cassandra
  • Preparou o ambiente para os próximos LABs
