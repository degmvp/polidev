Apache Cassandra – LAB 03
TTL (Time To Live)

Neste laboratório você aprenderá a utilizar TTL (Time To Live), recurso fundamental do Cassandra para expiração automática de dados.

Objetivo do LAB
  • Entender o conceito de TTL
  • Aplicar TTL em INSERT
  • Aplicar TTL em UPDATE
  • Consultar o tempo restante de vida

1️⃣ O que é TTL?

TTL define por quanto tempo um dado permanecerá armazenado no Cassandra. Após esse período, o registro é automaticamente marcado para remoção.

Uso comum de TTL
  • Logs
  • Sessões de usuário
  • Eventos temporários
  • Auditoria com retenção limitada

2️⃣ Criando tabela para logs

CREATE TABLE logs_acesso (
    usuario text,
    data_evento timestamp,
    evento text,
    PRIMARY KEY (usuario, data_evento)
);

3️⃣ INSERT com TTL

TTL é definido em segundos. 

INSERT INTO logs_acesso (usuario, data_evento, evento)
VALUES ('ana', toTimestamp(now()), 'LOGIN')
USING TTL 60;
Explicação:
O registro será removido automaticamente após 60 segundos.

4️⃣ Verificando TTL restante

SELECT usuario,
       evento,
       TTL(evento)
FROM logs_acesso
WHERE usuario = 'ana';

5️⃣ UPDATE com TTL

UPDATE logs_acesso
USING TTL 120
SET evento = 'LOGOUT'
WHERE usuario = 'ana'
AND data_evento = '2025-01-01 10:00:00';
Atenção:
TTL aplicado em UPDATE substitui o TTL anterior.

6️⃣ TTL em nível de coluna

É possível aplicar TTL apenas a uma coluna específica. 

UPDATE logs_acesso
USING TTL 30
SET evento = 'TIMEOUT'
WHERE usuario = 'ana'
AND data_evento = '2025-01-01 10:05:00';

7️⃣ Comportamento Interno

  • Dados expirados viram tombstones
  • Remoção física ocorre em compaction
  • TTL excessivo pode degradar performance

8️⃣ Boas Práticas

  • Use TTL para dados temporários
  • Evite TTL em dados permanentes
  • Planeje retenção de logs
  • Monitore tombstones

9️⃣ Encerramento do LAB

  • Você aplicou TTL com sucesso
  • Entendeu o ciclo de vida do dado
  • Preparou ambiente para índices
