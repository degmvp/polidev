# HACK25 — Privilege Escalation em Linux e Windows

**⚠️ AVISO IMPORTANTE**

Este artigo possui finalidade exclusivamente educacional e defensiva. As técnicas apresentadas devem ser utilizadas apenas em laboratórios, ambientes controlados ou durante avaliações de segurança previamente autorizadas. O uso indevido contra sistemas de terceiros pode configurar crime.

---

# 1. Introdução

Após obter acesso inicial a um sistema durante uma avaliação de segurança autorizada, uma das etapas mais importantes consiste em verificar se existem falhas que permitam aumentar o nível de privilégio do usuário atual.

Esse processo é conhecido como **Privilege Escalation (Escalada de Privilégios)** e representa uma das fases mais críticas de um Pentest, pois demonstra o impacto real de vulnerabilidades de configuração, permissões inadequadas ou softwares desatualizados.

Neste artigo conheceremos as principais técnicas de enumeração, ferramentas e boas práticas utilizadas em ambientes Linux e Windows.

---

# 2. Privilege Escalation no Linux

## Enumeração Básica

Antes de qualquer tentativa de escalada, é fundamental compreender o ambiente.

Comandos úteis:

```bash
whoami
id
sudo -l
cat /etc/passwd
cat /etc/shadow
uname -a
cat /etc/os-release
ls -la /home
```

A enumeração cuidadosa normalmente revela informações suficientes para direcionar toda a auditoria.

---

## Ferramentas Automatizadas

O **LinPEAS** é uma das ferramentas mais utilizadas para identificar possíveis vetores de escalada.

```bash
wget https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh

chmod +x linpeas.sh

./linpeas.sh
```

Outros comandos frequentemente utilizados durante a enumeração:

```bash
sudo -l | grep -v "permission denied"

find / -perm -4000 2>/dev/null

find / -writable 2>/dev/null
```

---

## Vetores comuns encontrados

* Binários SUID e SGID.
* Configurações incorretas no arquivo **sudoers**.
* Cron Jobs mal configurados.
* Serviços executando como root.
* Arquivos sensíveis com permissões inadequadas.
* Kernel desatualizado com vulnerabilidades conhecidas.

---

# 3. Privilege Escalation no Windows

## Enumeração Básica

Alguns comandos importantes para reconhecimento do ambiente:

```cmd
whoami
whoami /priv
systeminfo
net user
net localgroup administrators
powershell -c "Get-WmiObject -Class Win32_Product"
```

A enumeração é responsável por identificar privilégios disponíveis, grupos administrativos, serviços instalados e possíveis vetores de exploração.

---

## Ferramentas Automatizadas

### PowerUp

```cmd
powershell -ep bypass

. .\PowerUp.ps1

Invoke-AllChecks
```

### WinPEAS

```cmd
wget https://github.com/carlospolop/PEASS-ng/releases/latest/download/winPEASx64.exe

winPEASx64.exe
```

Essas ferramentas automatizam centenas de verificações que seriam demoradas manualmente.

---

## Vetores comuns encontrados

* SeImpersonatePrivilege.
* SeDebugPrivilege.
* Unquoted Service Path.
* AlwaysInstallElevated.
* Weak Service Permissions.
* DLL Hijacking.
* Scheduled Tasks.
* Kernel vulnerável e desatualizado.

---

# 4. Boas Práticas

* Sempre realize uma enumeração completa antes de qualquer validação.
* Prefira técnicas **Living off the Land (LotL)** sempre que possível.
* Documente cada evidência encontrada.
* Trabalhe exclusivamente em ambientes autorizados.
* Avalie o impacto da vulnerabilidade antes de qualquer tentativa de exploração.
* Registre versões do sistema operacional, patches e configurações relevantes.
* Valide todas as descobertas antes de emitir recomendações.

---

# Conclusão

A escalada de privilégios continua sendo uma das etapas mais importantes de uma avaliação de segurança. Em muitos casos, pequenas falhas de configuração permitem que um usuário com privilégios limitados obtenha acesso administrativo ao sistema.

Dominar técnicas de enumeração, compreender os principais vetores de escalada e utilizar ferramentas especializadas de forma responsável permite identificar riscos reais e fortalecer significativamente a segurança dos ambientes Linux e Windows.
