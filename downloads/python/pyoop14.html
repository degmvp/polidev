Módulo 13 - Testes

O que são Testes?

Testes são procedimentos que verificam se um software está funcionando conforme o esperado. Em POO, é essencial garantir que objetos e métodos se comportem corretamente.

Tipos de Testes

  • Testes Unitários: Verificam o comportamento de unidades isoladas (funções, métodos).
  • Testes de Integração: Avaliam a interação entre diferentes componentes.
  • Testes de Sistema: Validam o sistema como um todo.

Biblioteca unittest do Python

O Python possui a biblioteca unittest para facilitar a criação e execução de testes.

Exemplo de Teste Unitário

import unittest

class Calculadora:
def somar(self, a, b):
return a + b

class TestCalculadora(unittest.TestCase):
def test_somar(self):
calc = Calculadora()
self.assertEqual(calc.somar(2, 3), 5)

if __name__ == '__main__':
unittest.main()

Boas Práticas

  • Testar o maior número possível de casos.
  • Escrever testes antes do código (TDD - Test Driven Development).
  • Automatizar a execução de testes.

Resumo

Testar é essencial para garantir a qualidade, manutenção e evolução de sistemas orientados a objetos.
