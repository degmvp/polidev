Módulo 12 - Design Patterns

O que são Design Patterns?

Design Patterns (padrões de projeto) são soluções reutilizáveis para problemas comuns no desenvolvimento de software orientado a objetos.

Tipos comuns de Design Patterns

  • Singleton: Garante que uma classe tenha apenas uma instância.
  • Factory Method: Cria objetos sem expor a lógica de criação ao cliente.
  • Observer: Permite que objetos sejam notificados de mudanças em outros objetos.
  • Strategy: Define uma família de algoritmos intercambiáveis.

Exemplo: Singleton em Python

class Singleton:
_instance = None

def __new__(cls):
if cls._instance is None:
cls._instance = super().__new__(cls)
return cls._instance

# Teste
s1 = Singleton()
s2 = Singleton()
print(s1 is s2) # True

Exemplo: Factory Method simples

class Carro:
def dirigir(self):
pass

class CarroEsportivo(Carro):
def dirigir(self):
print("Dirigindo carro esportivo")

class CarroPopular(Carro):
def dirigir(self):
print("Dirigindo carro popular")

def criar_carro(tipo):
if tipo == "esportivo":
return CarroEsportivo()
elif tipo == "popular":
return CarroPopular()
else:
return Carro()

carro = criar_carro("esportivo")
carro.dirigir()

Resumo

Design Patterns ajudam a criar sistemas flexíveis, reutilizáveis e de fácil manutenção. Conhecê-los é essencial para programadores orientados a objetos experientes.
