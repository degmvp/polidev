Módulo 17 - Ambientes Virtuais

Importante: Ambientes virtuais são essenciais para isolar dependências entre projetos Python. Este módulo é fundamental para os próximos conteúdos.

1. Ambientes Virtuais no PyCharm

Criando ambiente virtual no PyCharm:

  1. Abra seu projeto no PyCharm
  2. Vá em File > Settings (Windows/Linux) ou PyCharm > Preferences (macOS)
  3. Navegue até Project: [nome_do_projeto] > Python Interpreter
  4. Clique na engrenagem e selecione "Add"
  5. Escolha "Virtualenv Environment"
  6. Selecione a localização e a versão do Python
  7. Marque "Make available to all projects" se desejar
  8. Clique em OK
# No terminal do PyCharm você pode verificar o ambiente ativo
which python
# ou
python --version

2. Navegando no Prompt de Comando

Comandos básicos para navegar entre diretórios:

cd nome_da_pasta       # Entra na pasta
cd ..                # Volta uma pasta
dir (Windows)        # Lista arquivos
ls (Linux/macOS)    # Lista arquivos
mkdir nova_pasta   # Cria nova pasta
pwd (Linux/macOS)   # Mostra caminho atual
Dica: No Windows, você pode usar o comando cd com a tecla TAB para autocompletar nomes de pastas.

3. Criando, Ativando e Removendo Ambientes Virtuais

Usando venv (módulo padrão do Python):

# Criar ambiente virtual (substitua 'meu_ambiente' pelo nome desejado)
python -m venv meu_ambiente

# Ativar ambiente (Windows)
meu_ambiente\Scripts\activate

# Ativar ambiente (Linux/macOS)
source meu_ambiente/bin/activate

# Desativar ambiente
deactivate

# Remover ambiente (apenas delete a pasta)
rm -rf meu_ambiente  # Linux/macOS
rd /s /q meu_ambiente  # Windows
Alternativa: Para projetos mais complexos, considere usar pipenv ou poetry que gerenciam ambientes e dependências juntos.

4. Ambientes Virtuais no VSCode

Configurando ambiente virtual no VSCode:

  1. Abra a pasta do projeto no VSCode
  2. Abra o terminal integrado (Ctrl+`)
  3. Crie o ambiente virtual como mostrado na seção 3
  4. Pressione Ctrl+Shift+P e digite "Python: Select Interpreter"
  5. Selecione o interpretador do ambiente virtual criado
  6. O VSCode geralmente detecta automaticamente ambientes virtuais na pasta do projeto
# Verificando o interpretador selecionado no VSCode
import sys
print(sys.executable)

Boas Práticas com Ambientes Virtuais

  • Crie um ambiente virtual para cada projeto
  • Nomeie os ambientes de forma descritiva (ex: projeto-analise-dados)
  • Não versione a pasta do ambiente virtual no Git
  • Use requirements.txt para listar dependências
  • Atualize regularmente os pacotes no ambiente

Gerenciando dependências

# Salvar dependências em requirements.txt
pip freeze > requirements.txt

# Instalar dependências de requirements.txt
pip install -r requirements.txt

# Listar pacotes instalados
pip list

# Atualizar todos os pacotes
pip freeze | %{$_.split('==')[0]} | %{pip install --upgrade $_}
