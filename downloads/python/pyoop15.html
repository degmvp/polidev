Módulo 14 - Análise e Projeto Orientado a Objetos

O que é Análise e Projeto OO?

É o processo de entender os requisitos do sistema e planejar sua estrutura usando os princípios da Programação Orientada a Objetos (POO).

Modelagem com UML

A UML (Unified Modeling Language) é a linguagem padrão para modelagem de sistemas orientados a objetos.

Principais Diagramas UML

  • Diagrama de Classes: Mostra as classes do sistema, seus atributos, métodos e relacionamentos.
  • Diagrama de Casos de Uso: Representa as funcionalidades esperadas do sistema do ponto de vista do usuário.
  • Diagrama de Sequência: Mostra a interação entre os objetos ao longo do tempo.
  • Diagrama de Atividades: Modela o fluxo de trabalho ou processos do sistema.

Exemplo de Classe em Código

class Cliente:
def __init__(self, nome, email):
self.nome = nome
self.email = email

def exibir_dados(self):
print(f"Cliente: {self.nome} - {self.email}")

Etapas do Projeto OO

  • Levantamento de Requisitos
  • Modelagem de Casos de Uso
  • Identificação de Classes e Objetos
  • Definição de Relacionamentos e Comportamentos
  • Refinamento e Implementação

Resumo

A análise e projeto orientado a objetos ajuda a construir sistemas robustos, reutilizáveis e de fácil manutenção, com base em boas práticas de modelagem e arquitetura.
