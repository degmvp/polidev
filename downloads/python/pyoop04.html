Módulo 3 - Encapsulamento

O que é Encapsulamento?

Encapsulamento é um dos pilares da Programação Orientada a Objetos (POO). Ele se refere à ocultação dos detalhes internos de um objeto, expondo apenas o que for necessário para seu uso externo.

Vantagens do Encapsulamento

  • Protege os dados do objeto contra acessos indevidos.
  • Permite controlar como os dados são acessados e modificados.
  • Facilita a manutenção e evolução do código.

Exemplo prático:

Vamos criar uma classe que representa uma conta bancária com encapsulamento.

class ContaBancaria:
def __init__(self, titular, saldo):
self.titular = titular
self.__saldo = saldo # atributo privado

def depositar(self, valor):
if valor > 0:
self.__saldo += valor
def sacar(self, valor):
if 0 < valor <= self.__saldo:
self.__saldo -= valor
else:
print("Saldo insuficiente.")
def ver_saldo(self):
return self.__saldo

# Testando a classe
conta = ContaBancaria("João", 1000)
conta.depositar(500)
print(conta.ver_saldo()) # saída: 1500
conta.sacar(300)
print(conta.ver_saldo()) # saída: 1200
print(conta.__saldo) # Erro! Atributo privado

Notação de Atributo Privado

Em Python, um atributo com dois underlines (__) no início é tratado como privado. Ele não deve ser acessado diretamente de fora da classe.

Resumo

Encapsulamento é uma prática essencial para proteger os dados internos de um objeto e garantir que seu uso externo seja feito de maneira segura e controlada. No próximo módulo, veremos como reutilizar código com Herança.
