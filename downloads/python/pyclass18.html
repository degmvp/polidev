Módulo 18 - Transformando Python em EXE

1. Executando Códigos em Python Automaticamente

Para agendar a execução automática de scripts Python:

No Windows (Agendador de Tarefas):

  1. Abra o Agendador de Tarefas
  2. Crie uma Tarefa Básica
  3. Defina o gatilho (diário, semanal, etc.)
  4. Ação: "Iniciar um programa"
  5. Programa/script: caminho para python.exe
  6. Argumentos: caminho para seu script.py
# Alternativa: usar o código Python com time.sleep()
import time
while True:
    # Seu código aqui
    time.sleep(86400)  # Executa diariamente (86400 segundos)

2. Instruções Importantes

ATENÇÃO:
  • Sempre teste seu código antes de converter para EXE
  • Mantenha uma cópia do código fonte
  • O executável será específico para o sistema operacional usado na compilação
  • Arquivos EXE podem ser flagrados como vírus por alguns antivírus

3. Python para EXE com Arquivos Simples

Usando PyInstaller (o método mais comum):

# Instale o PyInstaller
pip install pyinstaller

# Crie o executável (modo básico)
pyinstaller --onefile seu_script.py

# O executável estará em dist/seu_script.exe

Opções úteis do PyInstaller:

  • --onefile: Cria um único arquivo EXE
  • --windowed: Não mostra o console (para aplicações GUI)
  • --icon=seu_icone.ico: Adiciona ícone ao executável
  • --name=NomeDoApp: Define o nome do executável

4. Diminuindo Tamanho de Arquivos Executáveis

Técnicas para reduzir o tamanho do EXE:

# 1. Use UPX (compactador de executáveis)
pyinstaller --onefile --upx-dir=caminho_para_upx seu_script.py

# 2. Exclua bibliotecas não usadas
pyinstaller --onefile --exclude-module=tkinter seu_script.py

# 3. Use Python em modo otimizado
pyinstaller --onefile -O seu_script.py
Dicas adicionais:
  • Remova imports não utilizados no seu código
  • Considere usar --clean para remover caches entre builds
  • Para aplicações muito grandes, considere a abordagem de múltiplos arquivos

5. Adaptando o Código de Arquivos Complexos

Ajustes necessários para projetos complexos:

# 1. Tratamento de caminhos de arquivos (usar sys._MEIPASS)
import sys
import os

def resource_path(relative_path):
    """ Get absolute path to resource, works for dev and for PyInstaller """
    try:
        # PyInstaller creates a temp folder and stores path in _MEIPASS
        base_path = sys._MEIPASS
    except Exception:
        base_path = os.path.abspath(".")
    
    return os.path.join(base_path, relative_path)

# Uso:
image_path = resource_path('imagens/logo.png')

Para projetos com:

  • Arquivos de dados: Especifique no .spec ou use --add-data
  • Bibliotecas externas: Pode precisar de hooks personalizados
  • GUI (Tkinter, PyQt): Use --windowed e teste em máquinas limpas

6. Python para EXE com Arquivos Complexos

Processo para projetos com múltiplos arquivos e dependências:

# 1. Primeiro crie um arquivo .spec (opcional)
pyinstaller --onefile seu_script.py

# 2. Edite o arquivo seu_script.spec gerado
# Adicione arquivos de dados:
a = Analysis(
    ['seu_script.py'],
    binaries=[],
    datas=[('imagens/*.png', 'imagens'), ('dados/*.csv', 'dados')],
    hiddenimports=['pandas', 'numpy'],  # Para bibliotecas não detectadas
    ...
)

# 3. Recompile usando o arquivo .spec
pyinstaller seu_script.spec

Solução de Problemas Comuns

  • Executável não funciona em outra máquina:
    • Teste em uma máquina limpa
    • Inclua todas as dependências
    • Use --onefile apenas quando necessário
  • Erros de missing modules:
    • Use --hidden-import para módulos não detectados
    • Verifique se todas as bibliotecas estão instaladas
  • Executável muito grande:
    • Use as técnicas da seção 4
    • Considere dividir em múltiplos executáveis
