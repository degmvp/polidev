Mix-Adv 32 - Docstrings: Documentação de Funções

O que são docstrings?

Docstrings são cadeias de texto utilizadas para documentar funções, classes e módulos em Python. São escritas logo após a definição da função, entre três aspas (simples ou duplas).

Exemplo de docstring

def somar(a, b):
    """
    Retorna a soma de dois números.
    
    Parâmetros:
    a -- primeiro número
    b -- segundo número
    """
    return a + b

Como acessar a docstring

print(somar.__doc__)

Isso exibirá a documentação definida dentro da função.

Boas práticas para docstrings

  • Descreva claramente o propósito da função.
  • Indique os parâmetros e seus tipos, se necessário.
  • Explique o valor de retorno.
  • Use um estilo consistente.

Docstring de uma função com validação

def dividir(a, b):
    """
    Retorna a divisão de a por b.

    Lança uma exceção se b for zero.
    """
    if b == 0:
        raise ValueError("O divisor não pode ser zero.")
    return a / b

Documentação para classes

class Pessoa:
    """Representa uma pessoa com nome e idade."""
    
    def __init__(self, nome, idade):
        """
        Inicializa a pessoa com nome e idade.
        """
        self.nome = nome
        self.idade = idade

Resumo

  • Docstrings ajudam na manutenção e entendimento do código.
  • São acessíveis em tempo de execução via __doc__.
  • São diferentes de comentários normais (usam aspas triplas).
